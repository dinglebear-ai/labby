# CLI shims → shared dispatch + confirmation flags (v0.6.1)

## Session Overview
Pushed a staged refactor that rewires 16 CLI service commands from the legacy
`crate::mcp::services::<svc>::dispatch` path to the shared
`crate::dispatch::<svc>::dispatch` layer, and threads `-y/--yes` and
`--dry-run` flags through the common action helpers so destructive actions
have a uniform UX. Pushed straight to `main` as `9d1d355`.

## Behavior Changes (Before/After)
- Before: service CLI subcommands dispatched through `mcp::services::<svc>::dispatch`; no CLI-level `--dry-run` or `--yes` on these commands.
- After: same subcommands dispatch through `dispatch::<svc>::dispatch`; every command accepts `-y/--yes` (skip destructive confirmation) and `--dry-run` (print the action + params without executing).
- `action` parser now validates against the service's `ACTIONS` catalog at the clap layer, so unknown actions fail before dispatch.
